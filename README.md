# Lottery Prediction Helper

This repository contains a set of utilities to search, highlight and analyse 4-digit numbers across Excel datasets (podium.xlsx and family_tree.xlsx). It is interactive and designed to run from the project root.

## What it does
- Opens `podium.xlsx` and helps you select a target cell (4-digit number).
- Generates neighbours and families for the selected number using deterministic transformations.
- Highlights matching cells in `podium_highlighted.xlsx`, `family_tree_highlighted.xlsx` and `next_podiums.xlsx` using color-coding.
- Saves a text log of the session to `result.md`.

## Prerequisites
- Python 3.10 or newer
- The `openpyxl` package to read/write Excel files
- (Optional, recommended) LibreOffice or another spreadsheet application to view `.xlsx` files when the script opens them automatically. On Linux the script will fall back to `xdg-open` if LibreOffice is not available.

Install the Python dependency:

```bash
python -m pip install --upgrade pip
python -m pip install openpyxl
```

## Required files (place in repository root)
- `podium.xlsx` (required) — main dataset used by the script
- `family_tree.xlsx` (optional) — used by later steps when present

If these files are missing the script prints an error and exits.

## Run

1. Open a terminal in the project root and run:

```bash
python main.py
```

2. The script follows an interactive 14-step workflow (summarised):

- **Step 1 — User Cell Selection:** `podium.xlsx` is opened for reference and you are prompted to enter a cell coordinate (e.g., `Q1086`). The selected 4-digit number becomes the processing baseline.

- **Step 2 — Permutation Limitation Check:** The script computes unique permutations for the selected number. Processing continues only if the number has 12 or fewer permutations and exactly one repeating digit; otherwise you are asked to choose another cell.

- **Step 3 — Generating Neighbours:** Neighbours are generated by incrementing/decrementing each non-repeating digit (modulo 10), then sorting the digits descending.

- **Step 4 — Searching & Highlighting (podium):** The script searches `podium.xlsx` for these Neighbours restricted to the same company block and highlights matches in **light yellow**; the originally selected cell is highlighted in **light green**. Results are saved as `podium_highlighted.xlsx` and opened automatically.

- **Step 5 — Highlighted Cell Selection:** Review `podium_highlighted.xlsx` and enter the coordinate of one of the highlighted cells. The script validates the choice and closes the highlighted workbook before continuing.

- **Step 6 — Family Generation:** From the validated selection (value `B`) the script generates a Family of 4 numbers by adding 1 to each digit (mod 10) repeatedly and prints them.

- **Step 7 — Blocks for Family:** For each Family number the full 8-member Block (B, SB, POT, POT SB, FA, FA SB, FA POT, FA POT SB) is generated and added to a processing pool.

- **Step 8 — Generate Block for `B`:** The script also expands the block for the originally selected `B` and adds these members to the global pool.

- **Step 9 — Neighbours for the 40-member pool:** Neighbours are generated for every number in the 40-member pool (family blocks + B block) using the same neighbour rules.

- **Step 10 — 24-Permutation Removal Filter:** Any neighbour with 24 permutations (all digits unique) is removed from further consideration.

- **Step 11 — Secondary Cross-Dataset Highlighting:** The filtered neighbours are searched across columns D–R of `podium.xlsx` and matches are highlighted in **light red** only when they occur strictly "before" the selected block (above or left-in-same-row) and within the same company. Magnum and Singapore are treated as a partnership for matching.

- **Step 12 — Red-Cell Selection:** Enter the coordinate of one light-red highlighted cell. The selected cell must **not** belong to the same draw block as the Step 5 yellow cell (same-day constraint). Typing `done` or `exit` skips Steps 12–14 entirely. There is no auto-selection fallback. Once accepted, the script generates neighbours for that cell and closes `podium_highlighted.xlsx`.

- **Step 13 — family_tree.xlsx Search (optional):** If `family_tree.xlsx` exists, the script locates the Step 5 value (grey), identifies columns containing it, then searches only those columns for neighbours of the Step 12 red cell (highlighting matches in **light blue**). Results are saved as `family_tree_highlighted.xlsx`.

- **Step 14 — Colour-coded Highlights & Output:** Unique light-blue values are expanded into blocks and neighbours; matches found in `podium_highlighted.xlsx` are colour-coded (four colours) and saved as `next_podiums.xlsx`. Summaries of matches are printed to the terminal.

3. Files created/overwritten by the run:

- `podium_highlighted.xlsx`
- `family_tree_highlighted.xlsx` (only if `family_tree.xlsx` exists)
- `next_podiums.xlsx`
- `result.md` (session log)

Notes:
- If a save fails due to the file being open, close the file and press ENTER when prompted by the script.
- If files don't open automatically on your system, ensure `libreoffice` or `xdg-open` are available in PATH.

## Troubleshooting
- Missing dependency: install `openpyxl` as shown above.
- If files don't open automatically on your system, ensure `libreoffice` or `xdg-open` are available in PATH.
